Klucz SSH i podstawowe komendy Git’a

SSH

  1. Co to jest klucz SSH?
    Klucz SSH to klucz publiczny i prywatny, służący do bezpiecznego łączenia się ze zdalnymi serwerami. Klucz publiczny służy do kodowania informacji i może być przekazywany innym, natomiast klucz prywatny służy do ich odczytywania i musi być odpowiednio zabezpieczony, aby nie trafił w niepowołane ręce.
  2. Jak działa SSH?
    Działanie kluczy SSH oparte jest na tym, że są one w stanie wzajemnie się rozpoznawać. Zawsze funkcjonują w parach, umożliwiając bezpieczne połączenie ze zdalnym komputerem. Publiczny klucz z serwera weryfikuje obecność swojego prywatnego odpowiednika. Jeśli jest on poprawny, powinno nastąpić połączenie.
  3. Jak stworzyć parę kluczy SSH za pomocą Git Bash i jak dodać klucz SSH do konta GitHub?
    Podczas tworzenia kluczy SSH pierwszym krokiem powinno być sprawdzenie, czy przypadkiem taki klucz nie istnieje. Aby to wykonać, należy otworzyć Git Bash i wpisać poniższą komendę: 
    ls -al ~/.ssh
    Jeżeli wyświetli się błąd mówiący o nieistnieniu podkatalogu `.ssh` oraz związanych z nim plików, oznacza to, że nie posiadasz pary kluczy SSH w domyślnej lokalizacji i możesz je wygenerować.

    Aby wygenerować parę kluczy SSH należy użyć komendy: 
    ssh-keygen -t ed25519 -C "twoj_email@przyklad.com"
    Zamień twoj_email@przyklad.com na swój adres e-mail używany na GitHubie. Komenda utworzy nową parę kluczy SSH, wykorzystując podany adres jako etykietę.

    W trakcie generowania kluczy zobaczysz coś takiego: 
    Generating public/private ed25519 key pair.
Enter a file in which to save the key (/c/Users/nazwa_użytkownika/.ssh/id_ed25519):
    Naciśnij Enter, aby zapisać klucz w domyślnej lokalizacji. Albo możesz wpisać inną lokalizację w której chciałbyś zapisać klucz.

    Następnie program zapyta o hasło (opcjonalne dodatkowe zabezpieczenie). Jeśli nie chcesz ustawiać hasła, naciśnij dwa razy Enter: 
    Enter passphrase (empty for no passphrase):
Enter same passphrase again:

    Podczas wpisywania hasła nie będzie ono widoczne!!!

    Aby dodać klucz SSH do konta GitHub:
    1. Otwórz plik id_ed25519.pub w edytorze tekstowym (np. Notatnik).
    2. Skopiuj całą zawartość tego pliku.
    3. Przejdź na GitHub → Settings → SSH and GPG keys.
    4. Kliknij New SSH key, wpisz nazwę (np. „Mój laptop”) i wklej klucz w pole Key.
    5. Kliknij Add SSH key, aby zapisać.

Podstawowe komendy Git’a

  1. git init – Tworzy nowe repozytorium git w bieżącym katalogu.
  2. git status – Podaje wszystkie informacje na temat aktualnego stanu repozytorium.
  3. git add – Dodaje nowe lub zmienione pliki z katalogu roboczego do obszaru tymczasowego.

    4. Można dodać konkretny plik:

    git add nazwa_pliku
    lub dodać wszystkie nowe lub zmodyfikowane pliki: 
    git add .
  4. git commit – Zatwierdza dodane zmiany z komentarzem dotyczącym wykonanych modyfikacji.
    5. git commit -m "Komentarz dotyczący aktualizacji"
  5. git push – Wysyła lokalne zmiany do repozytorium zdalnego.
    6. git push origin main
  6. git pull – Pobiera zmiany z repozytorium zdalnego.
    7. git pull origin main
